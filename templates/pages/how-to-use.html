How to use

Django Todolist was created with Django and Django Rest Framework. It's a completed base for a todolist platform with API REST for develop apps and frontend app with your favorite technologies.

I use SQLite by default, because my intention is not to cause problems in your development, if you wanna learn Angular, React or Vue, you have everything to need for create a basic web or mobile app.

Authentication, basic crud, and basic change field state here.

If do you wanna try to learn django, this is a perfect way to learn all to do you need, ClassView, Login, Register and Logout, Rest Api, correct way to set html templates and assets, and external template framework for views.

I add a json collection for postman with all services, with this you do know how to use all services in REST API but, if you don't have a postman and you don't wanna install it, I'll try to describe this services here:

*domain*/auth/register/

Allow to create users. The params are username and password, the method is post.

*domain*/auth/login/

This service return an object with two properties, refresh and access. The access token must be send how an header authorization of type Bearer Token in all requests. The access token is alive by 5 minutes. If do you wanna make a request and the token is invalid, do you need to refresh the token.

*domain*/auth/refresh/

This service return an object with new access token if the old access token is invalid. The service needs the refresh token.

*domain*/tasks/

Return an array with all tasks for the user. The service needs the access token sent how Bearer Token.

In POST method, the request create a task with data included in the body form or Json Body

*domain*/tasks/*id*/

Return an object with the task info. The service needs the access token sent how Bearer Token.

  • In GET method, the request return task data.
  • In PUT method, the request modify the task data with the include body form or Json Body.
  • In DELETE method, the request delete the task.

Task Model Description

  • ID: The ID for task, generate with uuid.
  • Name: Name for task.
  • Content: The content for task.
  • User: Owner for this task, added in request in Django Views, and the token in API REST.
  • Status: Possible Status: todo, doing, inreview and done.
